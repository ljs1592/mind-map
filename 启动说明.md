# 🚀 启动说明

## 🎯 快速开始（3种方式）

### 方式 1: 重新构建并启动（推荐）⭐

**适用于**：第一次运行、修改了代码、或遇到问题时

**双击运行**: `rebuild-and-start.bat`

这会自动执行：
1. ✅ 检查 Node.js 环境
2. ✅ 安装依赖（如果需要）
3. ✅ 构建 simple-mind-map 库
4. ✅ 构建 web 应用
5. ✅ 启动开发服务器

### 方式 2: 快速启动（仅启动服务器）

**适用于**：已经构建过，只需要启动服务器

**双击运行**: `quick-start.bat` 或 `start-server.bat`

### 方式 3: 手动构建

```bash
# 1. 进入 web 目录
cd web

# 2. 安装依赖（首次或依赖变更时）
npm install

# 3. 构建库
npm run buildLibrary

# 4. 构建应用
npm run build

# 5. 返回根目录并启动
cd ..
node start-server.js
```

## ❌ 如果遇到问题

### 问题 1: 页面空白
**解决**: 
1. 查看控制台是否有错误
2. 确保服务器正在运行
3. 刷新页面（Ctrl + F5）

### 问题 2: 思维导图不显示
**解决**:
1. 检查是否通过 http://localhost 访问（不是 file://）
2. 确保构建成功（查看 simple-mind-map/dist 目录是否存在）
3. 清除浏览器缓存

### 问题 3: 勾选框不显示
**解决**:
1. 使用 **standalone-demo.html**
2. 检查控制台是否有 "TaskCheckbox 插件已加载" 信息
3. 确保数据包含 taskChecked 字段

### 问题 4: 控制台有错误
**解决**:
1. 截图发送错误信息
2. 确认使用的是 standalone-demo.html
3. 重新构建项目

## 📝 常见问题解决

### ❌ 问题：`vue-cli-service` 不是内部或外部命令

**原因**：没有安装依赖

**解决方案**：
```bash
cd web
npm install
```

或直接运行 `rebuild-and-start.bat`

### ❌ 问题：构建失败，提示缺少模块

**解决方案**：
```bash
cd web
# 删除旧的依赖
rm -rf node_modules
rm package-lock.json

# 重新安装
npm install
npm run buildLibrary
npm run build
```
